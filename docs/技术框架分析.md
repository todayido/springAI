# 技术框架分析

## 1. 核心框架
- **Spring Boot 2.7.18**: 作为基础框架，用于快速构建和部署应用程序。它简化了Spring应用的初始搭建以及开发过程。

## 2. 主要技术栈
- **Spring Web (spring-boot-starter-web)**: 用于构建Web应用程序，包括RESTful服务和MVC功能。
- **Spring Kafka (spring-kafka)**: 集成Apache Kafka，用于处理消息发布和订阅。
- **Java 8**: 应用程序的主要编程语言。
- **Maven**: 项目构建和依赖管理工具。

## 3. 应用结构
- **启动类**: `Application.java` 使用 `@SpringBootApplication` 注解，通过 `SpringApplication.run()` 启动应用。
- **控制器**: 
  - `HelloController.java` 提供主页和Kafka消息展示功能。
  - `KafkaTestController.java` 提供REST API用于发送Kafka测试消息。
- **服务类**: 
  - `KafkaConsumerService.java` 负责监听和处理Kafka消息。
  - `KafkaProducerUtil.java` 提供发送消息到Kafka的功能。
- **配置**: 
  - `application.properties` 包含服务器端口和Kafka相关配置。

## 4. 配置管理
- 应用配置集中在 `application.properties` 文件中，包括服务器端口和Kafka的消费者、生产者配置。

## 5. 测试框架
- 使用 **JUnit 5** 和 **Spring Boot Test** 进行单元测试和集成测试。
- 使用 **spring-kafka-test** 提供的工具进行Kafka相关的测试。

这个技术框架基于Spring Boot生态系统，结合了Web开发和消息队列处理能力，适合构建高性能的微服务应用。
